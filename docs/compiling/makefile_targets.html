Configuring Makefile targets

If your project is an OCaml Empty Makefile Project, you must tell the O'Caml plug-in what Makefile targets it must use when calling the make command to build your project, clean it, or create its documentation.

To do that, open the project properties by right-clicking on the project in the navigator and selecting Properties, and then click on the Makefile category on the left. You should get the following properties dialog:

There are three fields you can change:

  • rebuild target: specify the name of the makefile target that must be used to compile the project (usually "all")
  • clean target: specify the name of the makefile target that must be used to clean the project (usually "clean")
  • doc target: specify the name of the makefile target that must be used to generate documentation for the project

Any of these fields can contain several targets, separated by commas, that will be called in the order they appear.

The correct names of the targets that must be used can usually be found in a README or INSTALL file included with the project, or directly by analyzing the makefile itself.
