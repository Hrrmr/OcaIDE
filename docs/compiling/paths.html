Configuring project paths

Configuring the project paths is important, because the compiler and debugger rely on the paths to be correctly defined.

To open the project paths properties page, right-click on a project in the navigator, select Properties, and then open the Project Paths section.

Normally, the current "." folder and the O'Caml library's folder should be in this list. You should manually add all the folders in which the files used by your project can be found (this includes project sub-folders).

Each path can be either absolute or relative to the project.

The order of the paths is meaningful: if a file with the same name is found in two different folders, the first one will be taken.

If your project is an OCaml Makefile Project, you should also configure the paths in the Makefile Paths section, which are the paths in which the Makefile is going to look to find the tools it needs to compile your project. If you get a message like "command x not found" or similar in the Ocaml Compiler Output view while trying to compile a Makefile project, try to find the path of this command on your system and add it in the Makefile Paths properties page.
