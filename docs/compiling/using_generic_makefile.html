Using the generic Makefile

When you create an OCaml Makefile Project, two files are automatically created in this project: "makefile" and "OCamlMakefile". "OCamlMakefile" is a generic Makefile that knows how to compile complex projects, and "makefile" imports "OCamlMakefile", and is used to specify the list of files you want to compile, the compilation mode (bytecode, executable), along with some options (generate type annotations, ...).

You should modify "makefile" to reflect your project, whereas you normally would never modify "OCamlMakefile".

If you want to keep it simple, open "makefile" and just enter the names of the source files that must be compiled after "SOURCES =", in the right order, and enter the name of the executable you want to produce after "RESULT =".

If you want more information, refer to the official documentation for the OCamlMakefile project at http://www.ocaml.info/home/ocaml_sources.html#toc16.
