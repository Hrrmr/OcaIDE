Top-level

This OCaml plug-in for Eclipse includes an interface to the OCaml top-level, which is mainly used for testing code quickly. To open it, click on Window > Show View > OCaml Toplevel.

The top-level view is divided in two panels, which you can resize by moving the splitter which separates them. The top panel displays the output of the OCaml top-level, and the bottom panel is used to enter expressions.

To evaluate an expression, type it in the bottom panel, and press Enter (the expression must end with two semi-colons, and the text caret must be at the end of the expression when you type Enter).

You can edit text (using the arrow keys) before sending it to the top-level, unlike with the OCaml text-mode top-level.

You can also recall previously edited expressions by using the Up and Down arrow keys, or the F3 and F4 function keys when the expression spans several lines.

You can interrupt the computation at any moment by either clicking on the red button in the toolbar, or by hitting Ctrl + C (this does not work on Windows).

You can also use the toolbar buttons to clear the output, reset the top-level, or open a new top-level.

There is a shortcut which allows you to quickly send an expression from the code editor to the top-level. To activate it, select an expression in the OCaml code editor and click on OCaml > Eval In Toplevel or hit the F6 key (by default; this key can be changed in the preferences).

There is also a shortcut to load a compiled module into the top-level: right-click on the module (.ml file) in the navigator and select Load in Toplevel (note: this only works with byte-code modules).

Hint: Use topfind (from the findlib library) to easily load libraries into the top-level.
