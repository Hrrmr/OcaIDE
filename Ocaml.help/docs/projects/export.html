Exporting a project

If your project doesn't have a Makefile, you might want to create one before exporting it, so that it can be compiled easily outside of Eclipse.
To do that, right-click on your project, and select Generate Makefile in the pop-up menu (this step is optional):

Then, you can clean your project before exporting it, to get rid of useless files.
If it is checked, uncheck the Build Automatically option in the Project menu.
Then, click on Clean... in the Project menu.

You should get the following window:

Click on Clean projects selected below, and check only your project.
Uncheck the Start a build immediately checkbox, so that the project doesn't regenerate its files right after you cleaned them.
Then, click OK.

Now that your project is ready to export, right-click on it in the navigator view and select Export...

In the following dialog box, select File System under the General category, and click Next.

Then, you should get the following dialog box:

Check the boxes near the files you want to export, and uncheck those for the files you don't want to export. For example, you might want to uncheck the ".settings", "External files" and ".DebuggerSourceLookup" folders, as well as the ".paths", and ".project" files, which are used internally in the plug-in, but are not useful in an exported project.

Enter the path of the folder you want to export your project to (or click Browse... and select it), then click Finish.

Then, you can re-enable the Build Automatically checkbox you had disabled at the beginning.
